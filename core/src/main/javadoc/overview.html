ZeroAgent Framework

ZeroAgent is a modular Java 17+ framework for building agentic workflows using the OpenAI Chat Completions API. It provides a simple yet powerful abstraction layer for creating AI agents that can use tools, load skills from YAML, maintain memory across sessions, and execute complex reasoning loops.

Core Concepts

Agent

An Agent is the main orchestrator that manages conversations with the LLM. It handles the reasoning loop, executes tool calls, and maintains context across iterations. The {@link com.github.ndanhkhoi.zeroagent.agent.AgentLoop} is the heart of the framework, coordinating between the LLM, tools, and skills.

Tool

Tools are functions that the agent can call to perform actions or retrieve information. They are defined using the {@link com.github.ndanhkhoi.zeroagent.tool.Tool} interface and automatically converted to OpenAI function schemas. The {@link com.github.ndanhkhoi.zeroagent.tool.ToolRouter} manages tool registration and dispatches LLM function calls to the appropriate Java implementations.

Skill

Skills are reusable prompt templates defined in YAML format that can be dynamically loaded and injected into the agent's system prompt. The {@link com.github.ndanhkhoi.zeroagent.skill.SkillLoader} handles parsing, caching, and managing skills across sessions.

Memory

Memory provides persistence for agent conversations and state. The {@link com.github.ndanhkhoi.zeroagent.memory.Memory} interface defines the storage abstraction, with {@link com.github.ndanhkhoi.zeroagent.memory.InMemoryStore} providing a default in-memory implementation.

Getting Started

Basic Usage


// Create an LLM client
LlmClient llmClient = new OpenAiChatClient(apiKey);

// Create tools for the agent
List<Tool> tools = List.of(
    new CurrentTimeTool(),
    new JavaScriptTool()
);
ToolRouter toolRouter = new ToolRouter(tools);

// Configure the agent
AgentConfig config = new AgentConfig(
    "gpt-4o-mini",           // model
    0.7,                     // temperature
    10,                      // max iterations
    "You are a helpful assistant that can use tools and write code."
);

// Create and run the agent
AgentLoop loop = new AgentLoop(llmClient, toolRouter, config);
AgentResponse response = loop.run("What time is it? Write a Python function to calculate fibonacci numbers.");

Using Skills


// Load skills from YAML
SkillLoader skillLoader = new SkillLoader();
Map<String, Skill> skills = skillLoader.loadSkills("skills/");

// Inject skills into agent system prompt
String systemPrompt = skillLoader.injectSkills(config.systemPrompt(), skills);

AgentConfig configWithSkills = new AgentConfig(
    "gpt-4o-mini",
    0.7,
    10,
    systemPrompt  // Enhanced with skills
);

Memory Management


// Create a memory store
MemoryStore memoryStore = new InMemoryStore();

// Store conversation history
String sessionId = "user-session-123";
memoryStore.store(sessionId, userMessage, agentResponse);

// Retrieve conversation history
List<Message> history = memoryStore.retrieve(sessionId);

// Use in agent for context-aware conversations
AgentLoop loop = new AgentLoop(llmClient, toolRouter, config, memoryStore, sessionId);

Architecture

ZeroAgent follows a modular architecture with clear separation of concerns:

  • Agent Layer: {@link com.github.ndanhkhoi.zeroagent.agent} - Orchestrates the reasoning loop and manages LLM interactions
  • Tool Layer: {@link com.github.ndanhkhoi.zeroagent.tool} - Provides extensible function calling capabilities
  • Skill Layer: {@link com.github.ndanhkhoi.zeroagent.skill} - Enables dynamic prompt template management
  • LLM Layer: {@link com.github.ndanhkhoi.zeroagent.llm} - Abstracts LLM client interactions
  • Memory Layer: {@link com.github.ndanhkhoi.zeroagent.memory} - Handles conversation persistence
  • Concurrency Layer: {@link com.github.ndanhkhoi.zeroagent.concurrent} - Provides session-based locking for thread safety

Thread Safety

ZeroAgent provides thread safety guarantees through session-based locking:

  • {@link com.github.ndanhkhoi.zeroagent.concurrent.SessionLockManager} - Interface for session lock management
  • {@link com.github.ndanhkhoi.zeroagent.concurrent.PerSessionLockManager} - Default implementation using per-session locks

Each agent session is guaranteed to process messages sequentially, even when multiple requests arrive concurrently for the same session. Different sessions can process messages in parallel.
